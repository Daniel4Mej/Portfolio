AI-Powered Facial Recognition Door Handle

A smart lock I developed with Dublin Padaloy and Esau Portillo using a Raspberry Pi, machine learning libraries, and our own face encodings. It locks and unlocks using servo motor functions I integrated into a custom 3D-printed door handle. All face data is stored locally on the Pi with no Wi-Fi or cloud connection required, which makes it cheaper and more secure than most commercial smart locks.

Key Technical Features
—
Two-Factor Authentication

The lock needs both face recognition and a 2-second hold to actually trigger the servo. That window was important to add because without it, a photo or quick glance could set it off.

—
Fully Local — No Wi-Fi Required

Everything runs on the Pi. The face encodings are stored in encodings.pickle directly on the device, so there's no server, no cloud, and no internet dependency. That also makes it a lot harder to attack remotely compared to most commercial smart locks.

—
Custom 3D-Printed Hardware

We designed the doorknob and the Pi enclosure in CAD and 3D printed both as prototypes. Getting the servo to actually fit inside the doorknob mechanism and actuate cleanly took a few iterations.

—
Camera Retry Logic

If the camera doesn't initialize on the first try, the code retries up to 3 times and cleans up between each attempt. Without that the Pi would sometimes hold onto a dead resource and just hang.

Servo Control via GPIO 18

Using gpiozero with pulse widths calibrated to 0.5ms–2.5ms. The position range runs from −1 to 1, which maps directly to 0° through 180° on the motor for the lock and unlock positions.

Core Recognition Loop
import face_recognition
import cv2
import time
from collections import defaultdict

def process_faces_with_recognition_timer(run_time=30):
    picam2 = Picamera2()
    picam2.configure(picam2.create_preview_configuration(
        main={"format": 'XRGB8888', "size": (640, 480)}))
    picam2.start()
    face_recognized = 0
    face_timers = defaultdict(lambda: None)
    recognition_threshold = 2  # seconds

    while True:
        frame = picam2.capture_array()
        rgb = cv2.cvtColor(frame, cv2.COLOR_BGR2RGB)
        boxes = face_recognition.face_locations(rgb, model="hog")
        encodings = face_recognition.face_encodings(rgb, boxes)
        for box, encoding in zip(boxes, encodings):
            now = time.time()
            if face_timers[tuple(box)] is None:
                face_timers[tuple(box)] = now
            elif now - face_timers[tuple(box)] > recognition_threshold:
                face_recognized = 1  # TRIGGER UNLOCK
                face_timers[tuple(box)] = None

Servo Actuation
from gpiozero import Servo
from time import sleep

# GPIO 18, calibrated pulse widths
servo = Servo(18, min_pulse_width=0.5/1000,
                  max_pulse_width=2.5/1000)

def turn_servo_to(position):
    # -1 = 0deg | 0 = 90deg | 1 = 180deg
    servo.value = position
    sleep(1)

# Unlock sequence
turn_servo_to(0)   # center (locked)
turn_servo_to(1)   # rotate open
sleep(2)
turn_servo_to(0)   # return to locked

Deutsch-Jozsa Algorithm on Classical Machines

A paper I co-wrote that examines whether the Deutsch-Jozsa algorithm proves quantum superiority or if a classical computer can replicate the same result. Using Qiskit, we found that a classical computer can simulate the same results as a quantum computer, but only because it uses a transparent oracle rather than a true black-box, which changes what the result actually proves.

Read Research ↓
Quantum Computing Research Qiskit Python Algorithm Analysis
Abstract

The Deutsch-Jozsa algorithm is commonly cited as proof that quantum computers are fundamentally more powerful. We built a classical simulation in Qiskit and ran it for n<20. It produced the same results as the quantum version, but the reason it works classically is that the oracle is transparent — you can see the gates. The original quantum version assumes you can't, so it's not a straightforward comparison and the conclusion is more complicated than it looks.

Key Findings
—
Classical Simulation Works (n<20)

The simulation matched the quantum circuit result every time we tested it. Constant functions return all zeros and balanced functions return non-zero, both in a single query.

—
The Oracle Isn't Actually a Black Box

The only reason the classical simulation works is that the oracle has to be transparent, meaning the CNOT gates are visible. The original quantum algorithm assumes a true black-box oracle, so the two aren't actually doing the same thing.

—
Quantum Speed-Up Isn't That Clear-Cut

Johansson (2017) and Yavuz & Yadav (2023) independently reached the same conclusion. The Deutsch-Jozsa algorithm is a useful way to explain quantum concepts but it's not the definitive proof of quantum advantage that it's often treated as.
