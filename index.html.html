AI-Powered Facial Recognition Door Handle

A smart lock I built using a Raspberry Pi that recognizes faces in real time and uses a servo motor to unlock a door handle. I trained it on my own face encodings, and added a 2-second confirmation window so it doesn't trigger on a quick glance or a photo.

Key Technical Features
—
2-Second Confirmation Timer

The lock won't trigger unless the same face is held for 2 full seconds. Stops it from firing on a photo or a quick walk-by.

—
Pre-Encoded Face Store

Known faces are encoded ahead of time and saved to encodings.pickle. At runtime it just compares the live frame against that file using face_recognition.compare_faces().

—
Camera Retry Logic

If the camera fails to initialize it tries again up to 3 times, cleaning up the object between each attempt so the Pi doesn't hold onto a dead resource.

Servo Control via GPIO 18

gpiozero Servo with calibrated pulse widths (0.5ms–2.5ms). Position: −1 (0°) → 0 (90°) → 1 (180°) for precise lock/unlock.

Core Recognition Loop
import face_recognition
import cv2
import time
from collections import defaultdict

def process_faces_with_recognition_timer(run_time=30):
    picam2 = Picamera2()
    picam2.configure(picam2.create_preview_configuration(
        main={"format": 'XRGB8888', "size": (640, 480)}))
    picam2.start()
    face_recognized = 0
    face_timers = defaultdict(lambda: None)
    recognition_threshold = 2  # seconds

    while True:
        frame = picam2.capture_array()
        rgb = cv2.cvtColor(frame, cv2.COLOR_BGR2RGB)
        boxes = face_recognition.face_locations(rgb, model="hog")
        encodings = face_recognition.face_encodings(rgb, boxes)
        for box, encoding in zip(boxes, encodings):
            now = time.time()
            if face_timers[tuple(box)] is None:
                face_timers[tuple(box)] = now
            elif now - face_timers[tuple(box)] > recognition_threshold:
                face_recognized = 1  # TRIGGER UNLOCK
                face_timers[tuple(box)] = None

Servo Actuation
from gpiozero import Servo
from time import sleep

# GPIO 18, calibrated pulse widths
servo = Servo(18, min_pulse_width=0.5/1000,
                  max_pulse_width=2.5/1000)

def turn_servo_to(position):
    # -1 = 0deg | 0 = 90deg | 1 = 180deg
    servo.value = position
    sleep(1)

# Unlock sequence
turn_servo_to(0)   # center (locked)
turn_servo_to(1)   # rotate open
sleep(2)
turn_servo_to(0)   # return to locked

Deutsch-Jozsa Algorithm on Classical Machines

A research paper I co-wrote examining whether the Deutsch-Jozsa algorithm actually proves quantum superiority, or whether a classical computer can replicate the same result. We found that it can, but only because the oracle used in the classical version is transparent — a different assumption than the quantum version makes.

Read Research ↓
Quantum Computing Research Qiskit Python Algorithm Analysis
Abstract

The Deutsch-Jozsa algorithm gets cited a lot as proof that quantum computers are fundamentally better. We built a classical simulation in Qiskit and tested it for n<20. It worked — but only because the oracle is transparent, meaning you can see the gates. That's a different assumption than the original quantum version makes, which changes what the result actually proves.

Key Findings
—
Classical Simulation Works (n<20)

The simulation gets the same result as the quantum circuit every time — constant functions return all zeros, balanced functions return non-zero, in a single query.

—
The Oracle Isn't Actually a Black Box

To make it work classically, the oracle has to be transparent — you can see the CNOT gates. The original algorithm assumes you can't, so it's not a true apples-to-apples comparison.

—
Quantum Speed-Up Isn't That Clear-Cut

Johansson (2017) and Yavuz & Yadav (2023) both reached similar conclusions independently. D-J is a useful teaching tool but it's not the slam-dunk proof of quantum advantage people treat it as.
